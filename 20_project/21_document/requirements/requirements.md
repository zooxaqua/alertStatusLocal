# requirements 成果物

## 1. 目的
Step0（`--once`でSlack通知のみ）を先行実装し、ローカル cron 運用の価格監視ツールを段階的に構築する。

## 2. 入力
- 前工程成果物: `00_request/minutes_v000.md`
- 要求/制約:
  - new モード
  - ローカル cron 運用
  - 通知設定は `config.json` 集約
  - ブラウザ管理画面要件あり

## 3. 全体構成
- Step0: 通知疎通確認（通知のみ）
- Step1-4: 価格取得・最適化
- Step5: ブラウザ管理画面
- テスト: unit/integration/system を分離

## 4. 詳細
- 正常系:
  - `--once` 実行で通知成功
  - 監視対象管理が画面から実施可能
  - cron で定期実行される
- 異常系:
  - webhook 未設定時にエラー返却
  - URL/価格の不正入力を保存拒否
  - 取得失敗時にログと画面へエラー表示
- 要件ID:
  - REQ-001 Step0通知実行
  - REQ-002 通知設定の`config.json`集約
  - REQ-003 ローカル cron 定期実行
  - REQ-004 ブラウザ管理画面
  - REQ-005 テスト（正常/異常）

## 6. 課題・リスク
- Slack実環境接続はWebhook実値が必要。
- Playwrightのブラウザインストールが未実施環境では取得機能が制限される。

## 7. 次工程への引き継ぎ
- 基本設計でCLI/UI/設定/テスト構成を明確化する。
- Step0を最優先で実装順に反映する。
